# Project Architecture and Explanation

This document explains the "under-the-hood" workings of the **Chef Claude** application. It's meant to help other developers understand the key architectural decisions and libraries used.

## 1. Application Flow
1. **User Input:** The user adds ingredients to the \MainComponent.jsx\ input field.
2. **State Management:** The ingredients are held in a React \useState\ array.
3. **API Request:** When the user clicks "Get a recipe", a \etch\ request is made to \/api/recipe\ with the ingredients.
4. **Backend Proxy:** The Vite proxy forwards this request to a local Express.js server (\server.js\). The Express server securely attaches the Google AI API key, executes the call to the Gemini model, and returns a Markdown response.
5. **Display & Styling:** The React frontend receives the Markdown string and uses \eact-markdown\ and Tailwind Typography plugin to render it beautifully in \ClaudeRecipe.jsx\.

## 2. Secure API Integrations
A core goal of this project is to keep the API key completely safe.
- **Why?** Exposing API keys in frontend code (like Vite's \VITE_API_KEY\) allows anyone checking the network tab of their browser to steal it.
- **Solution:** We use a lightweight Express.js server that acts as a middleman. The frontend talks to the middleman, and the middleman securely talks to the AI provider.

## 3. Beautiful Markdown Rendering
AI models (like Gemini/Claude) do not return structured HTML—they return raw Markdown text (e.g., \# Recipe\, \* Ingredient\).

### \eact-markdown\
This library parses the raw markdown string sent by the AI and translates it directly into React components (headings, lists, bold text, etc.).

### \@tailwindcss/typography\
By default, Tailwind CSS strips away all standard browser styling, meaning \eact-markdown\'s generated lists and headings would look like plain unformatted text! 
To fix this, we installed the Tailwind Typography plugin and applied the \prose\, \prose-stone\, and \max-w-none\ classes. 
- **What it does:** The \prose\ plugin looks for raw unstyled HTML (generated by the markdown parser) and automatically layers gorgeous, pre-calculated typography scales, line-heights, and spacing margins onto it.

## 4. Folder Structure Overview
- \src/MainComponent.jsx\: The primary smart component that manages ingredient state and makes the fetch call to the backend.
- \src/IngredientList.jsx\: A presentational component mapping over the ingredients list.
- \src/ClaudeRecipe.jsx\: The presentational component wrapping \<ReactMarkdown>\ to convert the backend response text into UI.
- \server.js\: The minimal Express wrapper functioning as our secure proxy for the Google AI Studio request.
